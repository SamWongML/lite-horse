**Status:** SHIPPED
**Started:** 2026-04-23
**Shipped:** 2026-04-25
**Predecessor:** [v0.2-embed-and-evolve.md](v0.2-embed-and-evolve.md)
**Status ledger:** [../PROGRESS.md](../PROGRESS.md)

---

## Objective

Ship a **sophisticated, interactive-first** CLI at `litehorse`. Bare
`litehorse` drops the user into a persistent, streaming, tool-aware REPL
that delivers the same class of interaction as the existing webapp.
Scripted subcommands (`litehorse sessions list`, `litehorse skills
evolve …`) are the *secondary* surface — fully supported, but not the
primary UX.

This reverses v0.2's "no CLI as product surface" stance. Reasoning:

1. The webapp-embed story is intact; `lite_horse.api` is unchanged. The
   CLI is a second, equally first-class consumer of the same runtime.
2. Users want the agent in a terminal with streaming markdown, slash
   commands, tool-call approval, session resume, and cost meter — not a
   thin REPL. Hermes and openclaw both ship this; we match parity.
3. Scripted subcommands come for free once the same internal modules are
   wired through a Typer tree. CI / cron / scripting use those; humans
   stay in the REPL.

`litehorse-debug` is deleted at the end of this plan — fully replaced.

## Non-goals (explicit cuts)

- **Textual / full-screen TUI.** We use prompt_toolkit + rich. Textual
  is incompatible with the streaming-output pattern we want and would
  force a rewrite of the output pipeline.
- **Chat-platform adapter** (Telegram, Slack) — permanently out.
- **Daemon / gateway / RPC / MCP-server-hosted-by-CLI.** The REPL runs
  in-process and exits. Config-level MCP clients are already supported.
- **Companion web UI subprocess protocol** (hermes' `tui_gateway`). We
  have a webapp; it talks to `lite_horse.api` directly.
- **Interactive config wizard.** `litehorse config edit` opens `$EDITOR`.
- **Voice / TTS / speech input.**

### From 2026 Python-agent-REPL community (aider, llm, goose, Claude Code)
- **`click-default-group`** to route bare `litehorse` → REPL while
  keeping the named Typer subcommand tree. Proven by `simonw/llm`.
- `rich.live.Live(Markdown(buf), refresh_per_second=12)` — accumulate
  full buffer, re-render; don't try to diff-patch tokens.
- **Two-level Ctrl-C:** first press cancels running task via
  `asyncio.CancelledError`; second press within 2 s exits.
- **`multiline=True` + `enable_bracketed_paste=True`** on `PromptSession`

- `prompt_toolkit.shortcuts.radiolist_dialog` for bounded session

- **Permission ladder y/n/A/N** per tool, with session allow/deny-list.
  Modes: `auto | ask | read-only`. Flag `--permission`, slash
  `/permission`.
- Logging to **stderr only**; `RichHandler(console=Console(stderr=True))`.
- `--help` must return in <200 ms; heavy imports (openai, prompt_toolkit,
  rich) stay inside command bodies.

## Hard contract (binding for every implementation subagent)

Any subagent that cannot honor one of these must **stop and escalate**,
not invent a workaround.

### Entry point & dispatch
- `[project.scripts] litehorse = "lite_horse.cli.app:main"` in
  `pyproject.toml`.
- Dispatch uses **`click-default-group.DefaultGroup`** at the top level.
  Bare `litehorse` → `repl` command. `litehorse "prompt"` with positional
  arg → one-shot via the same `repl` command. `litehorse sessions list`
  routes to the named Typer subtree.
- Typer subtrees (`sessions`, `skills`, `cron`, `memory`, `config`,
  `doctor`, `logs`, `debug`) are attached to the Click group via
  `typer.main.get_command(typer_app)`.
- `litehorse-debug` script is removed in Phase 30. Both coexist until
  then.

### Library stack (additions to pyproject)
- `typer >= 0.12, < 1.0`
- `click-default-group >= 1.2.2, < 2`
- `prompt_toolkit >= 3.0.47, < 4`    # 3.0.47 fixes Windows bracketed paste
- `rich >= 13.7, < 15`               # already transitive via typer
- `pydantic-settings >= 2.2, < 3`
- `pexpect >= 4.9, < 5`              # **dev-only**, for REPL smoke test

No other runtime deps. `rich` is pinned explicitly (not via `typer[all]`).
No `pyperclip`, no `Pillow` — use OS subprocess for clipboard image.

### Async handler pattern
Every async command body uses the `@arun` decorator from `_async.py`.
The REPL loop is `async def` top-to-bottom; only `asyncio.run()` call is
inside `repl.loop.main_loop()`.

```python
from lite_horse.cli._async import arun

@app.command()
@arun
async def scripted_cmd(...) -> None: ...
```

### Output discipline
- **Human mode**: data → stdout via `Console()`; logs / progress → stderr
  via `Console(stderr=True)`. ANSI auto-stripped on pipe.
- **`--json` flag** (scripted only; REPL ignores): NDJSON to stdout,
  nothing else. Records:
  ```
  {"kind": "result",  "data": {...}}
  {"kind": "item",    "data": {...}}
  {"kind": "progress","stage": "...", "pct": 0.4}
  {"kind": "error",   "code": 3, "message": "..."}
  ```
- **REPL mode**: stdout is the rendered chat stream; stderr is debug
  logs gated by `LITEHORSE_DEBUG=1` or `/debug`. Never interleave.
- `--help` to stdout (explicit exception).

### Exit codes (IntEnum, documented)
```
0   OK
1   generic error
2   usage error (Click default)
3   config error
4   auth error (no OPENAI_API_KEY, invalid)
5   not-found (session / skill / proposal)
6   conflict (concurrent session lock)
7   io error (disk full, DB locked after retries)
130 SIGINT (one-shot only; REPL handles Ctrl-C in-loop)
143 SIGTERM
```

### Config cascade precedence
`flag > LITEHORSE_* env > ~/.litehorse/config.yaml > hardcoded default`.
`pydantic-settings` loads via `BaseSettings(env_prefix="LITEHORSE_",
env_file="~/.litehorse/.env")`. REPL slash commands that patch config
write via atomic YAML (temp-file + rename, 0o600 perms) — same primitive
as hermes' `save_config_value`.

### TTY detection & degradation
`_tty.py` computes at startup:
```
IS_STDIN_TTY  = sys.stdin.isatty()
IS_STDOUT_TTY = sys.stdout.isatty()
NO_COLOR      = "NO_COLOR" in os.environ or os.environ.get("TERM") == "dumb"
FORCE_COLOR   = "FORCE_COLOR" in os.environ
```
Decisions:
- `not IS_STDIN_TTY` → read stdin into buffer, pass as one-shot prompt,
  skip REPL.
- `not IS_STDOUT_TTY` → strip ANSI, no `rich.live`, plain print.
- `NO_COLOR and not FORCE_COLOR` → `Console(no_color=True,
  highlight=False)`; bottom toolbar disabled.

### Signal handling
- **REPL Ctrl-C (two-press):** first press cancels the in-flight agent
  turn (cancels the asyncio Task, prints `[cancelled]`, returns to
  prompt). Second press within 2 s exits with 0.
- **REPL Ctrl-D on empty input:** exits with 0.
- **One-shot Ctrl-C:** cooperative stop, flush session, exit 130.
- **SIGTERM (scripted / scheduler):** `loop.add_signal_handler` on Unix;
  `signal.signal` fallback on Windows.
- Never double-trap: install handlers **after** `PromptSession` is built,
  or intercept via prompt_toolkit `KeyBindings` at the application layer.

### Fast-path `--help`
`litehorse --help` and every subcommand's `--help` must return in
< 200 ms. Enforced by a test that runs `python -X importtime` and
asserts no `openai`, `prompt_toolkit`, or `rich` imports in the
call-chain triggered by `--help`. Heavy imports stay in function bodies.

### Testing
- Unit tests use `prompt_toolkit.input.create_pipe_input` +
  `DummyOutput` + `create_app_session` — assert dispatch / state, not
  rendering.
- Scripted commands use `typer.testing.CliRunner(mix_stderr=False)`.
- Exactly one pexpect smoke test per phase that touches REPL:
  `litehorse` boots to prompt, accepts `/help`, `Ctrl-D` exits cleanly.
  Skipped on CI where `pexpect` is unavailable (we mark it `slow`).
- Import-isolation test: `import lite_horse.api` must not transitively
  load `lite_horse.cli` or `lite_horse.evolve`.

## Locked CLI surface

### Interactive (default)
```
litehorse                       # REPL
litehorse "write a haiku"       # one-shot, print, exit
litehorse -c "follow up"        # one-shot in the most-recent session
litehorse -p "prompt"           # print-mode one-shot (plain, no color)
litehorse --session <key>       # REPL bound to an existing session
litehorse --resume              # REPL; opens session picker first
```

### Scripted subcommand tree (Typer)
```
litehorse sessions {list, show <key>, search <q>, end <key>, cleanup}
litehorse skills   {list, show <slug>, evolve <slug> [--days N]}
litehorse skills   proposals {list [<slug>], show <path>, approve <path>, reject <path>}
litehorse cron     {list, add <spec>, enable <id>, disable <id>, remove <id>, run-once <id>, scheduler}
litehorse memory   {show, clear}
litehorse config   {show, path, edit}
litehorse doctor
litehorse logs     {tail [-n N] [-f], path}
litehorse debug    share
litehorse version
litehorse completion install {bash|zsh|fish}
```

### Slash commands (REPL — alias map in parens)
```
/help, /h                       # command reference
/exit, /quit, /q                # exit
/clear, /cls                    # clear scrollback (Ctrl-L does same)
/new                            # new session (preserves model/permission)
/resume [<key>]                 # pick session or resume by key
/fork                           # branch current session to a new key
/compact                        # compress-as-consolidation now
/share                          # export current session → debug upload
/model [<name>]                 # show / switch model (picker if bare)
/permission [auto|ask|ro]       # show / switch permission mode
/debug [on|off]                 # toggle verbose logging
/verbose [off|new|all]          # tool-call display level
/usage, /cost                   # token + dollar meter
/skills                         # list activatable skills (opens picker)
/skill <slug>                   # hint-activate a skill for next turn
/cron [list|add|enable|disable] # cron CRUD without leaving REPL
/memory [show|clear]            # memory inspection
/attach <path|url>              # add file or URL to next turn
/paste-image                    # attach clipboard image (vision models)
/abort                          # same as Ctrl-C (cancel current turn)
/logs [N]                       # tail stderr log into pager
/config [<key>=<value>]         # show or patch config (atomic YAML)
/editor                         # compose next prompt in $EDITOR
```

### Key bindings (REPL)
```
Enter                  newline (multiline mode)
Meta-Enter / Esc Enter submit
Ctrl-C                 1st: cancel turn · 2nd (<2s): exit
Ctrl-D                 exit (empty input only)
Ctrl-L                 clear scrollback
Ctrl-R                 reverse-incremental history search
Ctrl-X Ctrl-E          edit buffer in $EDITOR (prompt_toolkit built-in)
Ctrl-O                 toggle tool-call expansion (last N calls)
Ctrl-P                 session picker overlay
Tab                    completion (slash commands + paths after @)
```

### Persistent bottom toolbar
`<model>  session:<key[:8]>  ctx:<used>/<max> (pct%)  $<cost>  [<mode>]`
Re-renders on each keystroke (prompt_toolkit `bottom_toolbar`). Hidden
during `rich.live.Live` streaming (documented limitation); a one-line
status line printed above the Live block bridges the gap.
